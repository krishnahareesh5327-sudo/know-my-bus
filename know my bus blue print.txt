Project Blueprint: "Know My Bus"
Client Vision: A frictionless, crowdsourced public transit tracking system.

1. Platform & Accessibility
Platform: Web-based (Browser). No app download required.

Target Audience: General public/Commuters.

Philosophy: Simple, effective, and automated.

2. The Data Engine (Crowdsourcing)
Source: Passenger mobile GPS data.

Processing: The system will average the data points from multiple passengers on the same bus to determine a single, accurate "Centroid" location for the bus icon.

Validation (Option C): Automated Route Matching. The system must compare passenger movement against known bus route polylines to automatically identify which bus they are on.

3. Key User Features
Live Map: Real-time visualization of bus icons moving along routes.

Search & Find: Search by bus number or destination; GPS-based "nearest stop" locator.

Dual Display (Option C): Provide both a live visual map and a calculated ETA (Estimated Time of Arrival) in minutes.

Route Geometry: Visual polylines showing the fixed path of each bus route.

4. Technical Requirements for Developer "X"
Backend: Real-time data processing (WebSockets or Firebase) to handle constant GPS pings.

Algorithm: Implementation of a "Speed + Path" filter to distinguish bus passengers from pedestrians or cars.

Optimization: Aggregation logic to ensure that 10 people on one bus do not appear as 10 separate icons.

Privacy: Ensure GPS data is used anonymously and only for the duration of the trip.
